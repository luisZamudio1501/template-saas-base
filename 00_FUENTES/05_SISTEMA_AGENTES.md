📄 DOCUMENTO MAESTRO — SISTEMA DE AGENTES (FÁBRICA SaaS)
________________________________________
1. 🧠 VISIÓN GENERAL DEL SISTEMA
Este sistema de agentes forma el núcleo operativo de la fábrica de aplicaciones SaaS.
Su función es:
•	transformar ideas en productos construibles 
•	asegurar consistencia en el desarrollo 
•	evitar errores de enfoque 
•	acelerar la producción 
•	mantener calidad mínima antes de salir al mercado 
El sistema está compuesto por 4 agentes principales:
1.	Product Strategist 
2.	SaaS Architect 
3.	Builder 
4.	QA / Tester 
Cada agente cumple un rol específico dentro del flujo de producción.
________________________________________
2. 🔁 FLUJO GENERAL DE TRABAJO
El sistema funciona como una cadena:
Idea → Product Strategist → SaaS Architect → Builder → QA / Tester
Cada agente:
•	recibe una entrada definida 
•	realiza un proceso específico 
•	entrega una salida estructurada 
Ningún agente trabaja sin contexto.
Ningún agente reemplaza al anterior.
________________________________________
3. 🧠 AGENTE 1 — PRODUCT STRATEGIST
🎯 Misión
Transformar una idea vaga en un producto SaaS claro, simple y vendible (MVP).
________________________________________
📥 Entrada
•	idea cruda 
•	objetivo del producto 
•	contexto del negocio (si existe) 
•	restricciones 
________________________________________
⚙️ Proceso
1.	limpiar la idea 
2.	identificar el valor principal 
3.	definir el MVP real 
4.	eliminar funcionalidades innecesarias 
5.	definir flujo del usuario 
6.	estructurar módulos funcionales 
________________________________________
📤 Salida
Debe incluir:
•	definición del producto 
•	problema que resuelve 
•	público objetivo 
•	MVP (qué incluye / qué no incluye) 
•	flujo del usuario 
•	módulos funcionales 
•	riesgos o dudas 
________________________________________
📏 Reglas
•	no agregar complejidad 
•	priorizar velocidad de salida 
•	pensar en venta, no en perfección 
•	eliminar todo lo innecesario 
________________________________________
⏱️ Momento de uso
Siempre antes de cualquier diseño o desarrollo.
________________________________________
4. 🧱 AGENTE 2 — SAAS ARCHITECT
🎯 Misión
Convertir el MVP en una arquitectura técnica clara, modular y construible.
________________________________________
📥 Entrada
•	salida del Product Strategist 
________________________________________
⚙️ Proceso
1.	traducir módulos funcionales a módulos técnicos 
2.	definir estructura por capas (Pantallas → Servicios → Motor) 
3.	decidir nivel del sistema (Nivel 1 o Nivel 2) 
4.	definir contratos de datos 
5.	establecer límites del sistema 
________________________________________
📤 Salida
Debe incluir:
•	mapa de módulos técnicos 
•	estructura por capas 
•	stack recomendado 
•	contratos de datos 
•	flujo técnico 
________________________________________
📏 Reglas
•	no modificar el MVP 
•	no agregar funcionalidades 
•	mantener desacople 
•	evitar sobreingeniería 
________________________________________
⏱️ Momento de uso
Después del Product Strategist, antes del desarrollo.
________________________________________
5. ⚙️ AGENTE 3 — BUILDER
🎯 Misión
Convertir la arquitectura en un producto funcional.
________________________________________
📥 Entrada
•	salida del SaaS Architect 
________________________________________
⚙️ Proceso
1.	generar estructura del proyecto 
2.	crear módulos definidos 
3.	implementar pantallas 
4.	implementar servicios 
5.	conectar con el motor de datos 
6.	construir funcionalidades paso a paso 
________________________________________
📤 Salida
•	código funcional 
•	estructura del proyecto 
•	funcionalidades implementadas 
•	instrucciones ejecutadas 
________________________________________
📏 Reglas
•	no rediseñar el producto 
•	no modificar arquitectura 
•	construir por etapas 
•	mantener coherencia con el diseño 
________________________________________
⏱️ Momento de uso
Después del SaaS Architect.
________________________________________
6. 🧪 AGENTE 4 — QA / TESTER
🎯 Misión
Validar que el producto construido cumple el MVP y funciona correctamente.
________________________________________
📥 Entrada
•	producto construido 
•	definición del MVP 
•	flujo esperado 
•	arquitectura (opcional) 
________________________________________
⚙️ Proceso
1.	validar funcionalidad 
2.	comparar contra el MVP 
3.	testear flujo completo 
4.	detectar errores y riesgos 
________________________________________
📤 Salida
Debe incluir:
•	estado del producto 
•	lista de errores 
•	ajustes necesarios 
•	evaluación del flujo del usuario 
________________________________________
📏 Reglas
•	no rediseñar el producto 
•	no agregar funcionalidades 
•	enfocarse en funcionamiento real 
•	evaluar como usuario final 
________________________________________
⏱️ Momento de uso
Después del Builder, antes del lanzamiento.
________________________________________
7. 🔗 RELACIÓN ENTRE AGENTES
Cada agente depende del anterior:
•	el Architect depende del Strategist 
•	el Builder depende del Architect 
•	el QA depende de todos 
Si se rompe esta cadena:
→ el sistema pierde coherencia
________________________________________
8. 🧭 PRINCIPIOS DEL SISTEMA
•	no saltar agentes 
•	no construir sin definición 
•	no diseñar sin MVP 
•	no lanzar sin validación 
•	no mezclar roles 
________________________________________
9. 👤 ROL DEL USUARIO (DIRECTOR)
El usuario cumple funciones clave:
•	iniciar el proceso con una idea 
•	invocar cada agente 
•	revisar y aprobar salidas 
•	pasar información entre agentes 
•	ejecutar la construcción 
•	decidir cuándo avanzar 
________________________________________
10. 📌 DEFINICIÓN FINAL
Este sistema de agentes no reemplaza al desarrollador.
Permite:
•	trabajar con orden 
•	reducir errores 
•	acelerar decisiones 
•	convertir ideas en productos reales 
•	mantener coherencia en toda la fábrica
