# 📄 07_CONTROL_DE_DOCUMENTOS.md

## 🎯 OBJETIVO

Definir qué debe contener cada documento del proyecto para asegurar consistencia y evitar desviaciones.

Este documento es una referencia de control.

No reemplaza a los agentes.

---

## ⚠️ REGLA PRINCIPAL

Los documentos de `01_DOCUMENTOS`:

- NO deben ser completados manualmente
- deben ser generados por agentes
- este documento solo sirve para validar

---

## 📄 01_PRODUCT_STRATEGY.md

Debe contener:

- definición del producto
- problema que resuelve
- público objetivo
- MVP (incluye / no incluye)
- flujo del usuario
- módulos funcionales
- riesgos o dudas

Generado por:

👉 Product Strategist

---

## 📄 02_ARCHITECTURE.md

Debe contener:

- decisión de nivel (1 o 2)
- stack recomendado
- módulos técnicos
- estructura por capas
- contratos de datos
- flujo técnico

Generado por:

👉 SaaS Architect

---

## 📄 03_BUILD.md

Debe contener:

- pasos ejecutados
- módulos implementados
- decisiones técnicas
- problemas encontrados
- soluciones aplicadas

Generado por:

👉 Builder

---

## 📄 04_QA.md

Debe contener:

- estado del producto
- validación contra MVP
- validación del flujo
- errores detectados
- ajustes necesarios
- recomendación final

Generado por:

👉 QA / Tester

---

## 📄 05_BACKLOG.md

Debe contener:

- tareas pendientes
- mejoras futuras
- ideas no implementadas

---

## 📄 06_DECISIONES.md

Debe contener:

- decisiones importantes
- cambios de criterio
- cosas descartadas
- reglas específicas del producto

---

## 🚫 PROHIBIDO

- escribir estos documentos sin usar agentes
- completar parcialmente sin validación
- usar estos documentos como notas personales
- mezclar contenido de distintos agentes

---

## ✅ USO CORRECTO

Este documento se usa para:

- validar contenido generado
- detectar faltantes
- asegurar calidad mínima

No se usa para:

- escribir el producto
- reemplazar el flujo de agentes
